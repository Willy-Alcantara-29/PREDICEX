# Ramas para GitHub

## Ramas principales

- `main`: contiene versiones estables y listas para entregar.
- `develop`: integra el trabajo del equipo antes de pasarlo a `main`.

## Ramas de trabajo

- `feature/*`: nuevas funcionalidades.
- `docs/*`: cambios de documentacion.
- `fix/*`: correcciones.

## Recomendacion de uso

1. Crear ramas nuevas desde `develop`.
2. Hacer commits pequenos y descriptivos.
3. Abrir pull requests hacia `develop`.
4. Pasar de `develop` a `main` solo cuando la entrega este validada.
